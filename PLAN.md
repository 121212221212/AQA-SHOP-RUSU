# План автоматизации тестирования веб-сервиса покупки тура

## 1. Перечень автоматизируемых сценариев

### Позитивные сценарии:
-  **Оплата по дебетовой карте со APPROVED карты (1111 2222 3333 4444)**
    - Ожидаемый результат: Покупка успешна, сообщение "Операция одобрена Банком"

-  **Оплата по дебетовой карте с DECLINED карты (5555 6666 7777 8888)**
    - Ожидаемый результат: Покупка отклонена, сообщение "Ошибка! Банк отказал в проведении операции"

-  **Кредит по данным карты с APPROVED карты (1111 2222 3333 4444)**
    - Ожидаемый результат: Кредит одобрен, сообщение "Операция одобрена Банком"

-  **Кредит по данным карты с DECLINED карты (5555 6666 7777 8888)**
    - Ожидаемый результат: Кредит отклонен, сообщение "Ошибка! Банк отказал в проведении операции"

### Негативные сценарии (будут добавлены):
-  Оплата с невалидным номером карты
-  Оплата с истекшим сроком действия карты
-  Оплата с неверным CVC кодом
-  Оплата с пустыми обязательными полями

## 2. Перечень используемых инструментов с обоснованием выбора

- **Java 11** - основной язык разработки автотестов
- **JUnit 4** - фреймворк для написания тестов
- **Selenide** - фреймворк для UI-тестирования (удобный API, автоматическое управление браузером)
- **Gradle** - система сборки проекта
- **Allure** - фреймворк для построения отчетов
- **Docker** - для развертывания тестовых баз данных
- **MySQL/PostgreSQL** - СУБД для проверки сохранения данных
- **GitHub** - хостинг проекта и система контроля версий

## 3. Перечень и описание возможных рисков при автоматизации

1. **Нестабильность тестов** - Из-за timing issues или изменений в UI
2. **Проблемы с окружением** - Отсутствие Docker, проблемы с виртуализацией
3. **Баг в тестируемом приложении** - Некорректное отображение уведомлений
4. **Изменение API банковских сервисов** - Могут измениться эндпоинты или форматы запросов
5. **Ограничения доступа** - Нет доступа к логам банковского сервиса
