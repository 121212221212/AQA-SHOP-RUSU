# Отчет о тестировании веб-сервиса покупки тура

## Краткое описание

Проведено автоматизированное тестирование веб-сервиса покупки тура, который предлагает два способа оплаты:
1. **Обычная оплата** по дебетовой карте
2. **Выдача кредита** по данным банковской карты

Тестирование проводилось с использованием предопределенных тестовых карт:
- **APPROVED карта**: 1111 2222 3333 4444
- **DECLINED карта**: 5555 6666 7777 8888

## Количество тест-кейсов

Всего автоматизировано: **4 тест-кейса**

### Разбивка по функциональности:
- **Оплата по дебетовой карте**: 2 теста
- **Кредит по данным карты**: 2 теста

## Результаты тестирования

| Категория тестов | Всего тестов | Успешных | Неуспешных | % успеха |
|------------------|--------------|----------|------------|----------|
| Дебетовая карта  | 2 | 2 | 0 | 100% |
| Кредитная карта  | 2 | 2 | 0 | 100% |
| **Итого** | **4** | **4** | **0** | **100%** |

## Обнаруженные баги

###  Критический баг: Некорректное отображение уведомлений
- **Описание**: При успешной операции банка приложение показывает одновременно два уведомления - "Операция одобрена Банком" и "Ошибка! Банк отказал в проведении операции"
- **Шаги воспроизведения**:
    1. Заполнить форму оплаты с APPROVED картой (1111 2222 3333 4444)
    2. Нажать "Продолжить"
    3. Наблюдать два одновременно отображаемых уведомления
- **Ожидаемый результат**: Должно отображаться только одно уведомление "Операция одобрена Банком"
- **Фактический результат**: Отображаются оба уведомления одновременно
- **Влияние**: Пользователь видит противоречивую информацию о результате операции

###  Ограничение тестирования
- **Описание**: Нет доступа к логам банковского сервиса для проверки реальных ответов
- **Влияние**: Нельзя напрямую проверить корректность взаимодействия с банковским API
